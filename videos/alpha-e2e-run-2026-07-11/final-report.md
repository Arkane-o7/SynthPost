The alpha run completed end to end through the web UI: project creation, topic entry, research, hosted-AI script generation, visual acquisition and quarantine, timeline approval, preview, real Rocketbox avatar rendering, 1080p composition and episode assembly.

## Production quality

- The anchor and branded lower-third are visually consistent across sampled frames.
- Narration remains present over the fullscreen still image.
- Competing broadcaster packages were kept out of the edit.
- The program is presenter-heavy because the rights, resolution and cleanliness gates correctly rejected most web-video leads.
- One attributed third-party infographic is synthetic-looking. It is not generated by SynthPost, but it does not meet the desired documentary-photo taste. This should be addressed with an explicit synthetic/illustration aesthetic policy rather than weakening source-safety gates.

## Defects found and fixed

1. Groq rate limiting blocked a 10-minute script. Production now uses an explicit Groq-to-Gemini hosted fallback and a 45-second Groq network timeout.
2. Visual acquisition progress remained at the AI-planning label during downloads and OCR. The pipeline worked, but the UI still needs substage progress events.
3. The 1920x1080 input gate starved the edit. It now accepts 1366x768-or-better landscape media while rendering the program at 1080p.
4. Long-form avatar rendering exhausted disk space because old PNG capture directories were retained. The renderer now checks free space before starting and automatically removes its temporary capture directory.
5. A shared local-media folder could contaminate unrelated productions. Local media is now isolated by project and episode, and external paths are imported before processing.
6. A completed selected story could disappear behind the 100-candidate inbox limit. Selected candidates are now ordered first.

## Media isolation design

`projects/<project_id>/episodes/<episode_id>/media_inbox/` is the only local discovery root for an episode.

- `uploads/<story_id>/` contains browser uploads.
- `imports/<story_id>/` contains copies of files staged from arbitrary paths.
- Render-ready derivatives remain story-specific under `episodes/<episode_id>/stories/<story_id>/visuals/`.
- SearXNG downloads remain story-specific and never enter another project or episode's scan.
- The legacy global `media_drop` folder was migrated and removed.

## Alpha verdict

The complete production path works and produces a technically valid 1080p news program. It is ready for continued alpha testing, but not yet editorially autonomous: visual coverage is still sparse under strict rights rules, progress reporting is too coarse during acquisition, and synthetic-looking third-party imagery needs its own taste classifier.
